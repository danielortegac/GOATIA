1. Introducción y Compromiso

Goatify APP ("Nosotros", "La Plataforma") se compromete a proteger su privacidad con un enfoque de "Security First". Esta política detalla exhaustivamente la recopilación, uso y protección de la información. Al utilizar Goatify, usted acepta las prácticas descritas en este documento legal.

2. Datos que Recopilamos

Para el funcionamiento óptimo de los Agentes IA y el Gestor de Proyectos, recopilamos estrictamente lo necesario:

Cuenta

Nombre, Email, Foto (Google Auth).

Operativos

Proyectos, tareas y notas.

Integraciones

Técnicos

3. Uso de Permisos de Google (Scopes)

Goatify solicita permisos específicos ("scopes") para automatizar su agenda:

Permiso Propósito
.../auth/userinfo.email Autenticación única (SSO) y cuenta segura.
.../auth/calendar.events Permite a los Agentes IA agendar reuniones en tu nombre y detectar conflictos.
.../auth/calendar.readonly Lectura de disponibilidad para consultas rápidas antes de agendar.

Cumplimiento con la Política de Google API

El uso de Goatify APP y la transferencia a cualquier otra aplicación de la información recibida de las API de Google se adherirá estrictamente a la Política de Datos de Usuario de los Servicios de API de Google, incluidos los requisitos de Uso Limitado.

"Goatify's use and transfer to any other app of information received from Google APIs will adhere to Google API Services User Data Policy, including the Limited Use requirements."

7. Data Usage, Storage, Sharing & Deletion (Google User Data)

Permisos y Consentimiento (OAuth)

Cuando conectas tu cuenta de Google, Goatify solicita permisos a través de la pantalla oficial de consentimiento de Google. Tú decides si aceptas o rechazas esos permisos. Si no los aceptas, la integración con Google no se activa, pero la app sigue funcionando.

  • calendar.readonly: Para leer disponibilidad y evitar conflictos.
  • calendar.events: Para crear eventos solo bajo tu petición.

Puedes revocar el acceso en cualquier momento desde tu cuenta Google: Seguridad > Acceso de terceros.

Esta sección cumple específicamente con la Google API Services User Data Policy. Aquí explicamos con claridad cómo usamos, almacenamos, compartimos, protegemos y eliminamos datos que provienen de Google.

7.1 Data Usage (Uso de datos)

  • Inicio de sesión: Usamos tu email y foto de perfil para autenticarte.
  • Google Calendar (lectura): Consultamos disponibilidad para evitar choques de agenda.
  • Google Calendar (escritura): Creamos eventos únicamente cuando tú lo solicitas verbalmente o por texto a la IA.
  • NO Publicidad: No usamos tus datos de Google para publicidad ni perfiles de marketing.
Principio de Mínimo Acceso

Solicitamos solo los permisos estrictamente necesarios. Si no conectas tu cuenta Google, Goatify funciona sin acceder a datos de Calendar.

7.2 Data Sharing (Compartición de datos)

Compartimos datos únicamente con proveedores de infraestructura necesarios para operar, bajo estrictos contratos de confidencialidad:

Proveedor Rol Propósito
Google APIs Sub-procesador Lectura/Escritura de eventos.
Google Cloud / AWS Infraestructura Base de datos y alojamiento seguro.
OpenAI Enterprise Procesador IA Procesamiento de texto (Zero-Retention).

* No vendemos datos a corredores de datos ni redes publicitarias.

7.3 Data Storage & Protection

  • Almacenamiento: Guardamos datos en servidores seguros (Google Cloud/AWS) con controles de acceso estrictos.
  • Seguridad: Todo el tráfico está cifrado vía HTTPS/TLS 1.3. Los datos en reposo están cifrados (AES-256).

7.4 Data Retention & Deletion

  • Retención: Retenemos tus tokens mientras mantengas la integración activa.
  • Eliminación: Si eliminas tu cuenta o revocas el acceso, borramos tus tokens y datos asociados en un plazo máximo de 7 días.
Cómo pedir borrado de datos

Puedes solicitar la eliminación total desde la app (Configuración > Cuenta > Eliminar) o escribiendo a info@goatify.app.

7.5 AI/ML Model Training

No utilizamos tus datos de Google Workspace (Calendar, Email) para entrenar modelos de Inteligencia Artificial públicos o de terceros. El uso de la IA se limita a procesar tu solicitud específica en tiempo real.
