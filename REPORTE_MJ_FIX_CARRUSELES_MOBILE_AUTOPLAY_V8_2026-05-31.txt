REPORTE MJ — FIX CARRUSELES MOBILE AUTOPLAY V8
Fecha: 2026-05-31

Problema encontrado:
- En index.html había parches móviles contradictorios: un bloque apagaba el movimiento con animation:none y otro intentaba reactivarlo con CSS animation.
- En iPhone/Safari eso podía dejar el carrusel del Main quieto o inconsistente.
- En pricing/index.html los carruseles de planes estaban configurados para deslizarse manualmente, pero no tenían autoplay real.

Cambios aplicados:
1. index.html
- Se reemplazaron los parches conflictivos por un único bloque: goatify-main-mobile-carousel-fix-v8.
- Se conservó el diseño actual del hero y sus ajustes móviles.
- Se mantuvieron los highlights estratégicos del párrafo principal.
- Se añadió autoplay real por requestAnimationFrame para:
  a) .goat-visual-track dentro de .goat-visual-rail
  b) .brand-track dentro de .brand-marquee
- En móvil el JS controla el movimiento con transform: translate3d, evitando el choque de animation:none / animation CSS.
- El movimiento se pausa brevemente cuando el usuario toca o interactúa.

2. pricing/index.html
- Se reemplazó el script de swipe manual por goatify-pricing-mobile-autoplay-carousel-v8.
- Se mantuvo el swipe manual.
- Se añadió autoplay real para los carruseles de planes:
  a) #automation-plans-container
  b) #web-plans-container
  c) #ai-plans-container
- Se añadió también autoplay robusto para el carrusel visual superior .pricing-visual-track.
- El autoplay se pausa cuando el usuario toca, arrastra o interactúa.

3. Cache
- Se actualizó el service worker y el registro:
  goatify-cache-v20260531-carousel-autoplay-v8
- Esto evita que el celular siga cargando una versión vieja del sitio.

No se cambió:
- Textos comerciales.
- Links.
- Formularios.
- Diseño de escritorio.
- Secciones internas fuera de los carruseles solicitados.
- Portafolio, agenda, footer o cards no relacionadas.

Validación técnica:
- Se verificó sintaxis JS con node --check en ambos scripts nuevos.
- Se confirmó estructura HTML de los carruseles principales:
  index.html: .goat-visual-track tiene 20 cards y .brand-track tiene 40 items duplicados para loop infinito.
  pricing/index.html: .pricing-visual-track tiene 12 cards duplicadas para loop infinito.
